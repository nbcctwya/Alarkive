# 开发约定

- 页面路由只做查询、404 处理和组件组合。
- Client Component 不导入 `@/server/**`。
- 跨客户端与服务端的数据结构放在 `src/types/`。
- Repository 只访问 SQLite；跨数据库和文件系统操作放在 Service。
- 优先使用清晰的直接导入，避免创建层层 barrel exports。
- 一个文件应有一个清晰主题；页面协调器不应同时定义全部弹窗和子区域。
- 新抽象必须有当前使用场景，不为可能出现的需求预建框架。
- 用户可见错误必须显示在界面中，不能只写 `console.error`。
- 正式页面不得回退 seed 数据。
