# Reader 页面规格

## 页面目标

以接近教材的版式提供长文阅读体验，让读者掌握整书位置、本章结构与阅读进度。

## 页面区域与组件层级

- `ReaderHeader`：品牌、文档标题、总进度、返回 Library、进入编辑。
- `ReaderSettings`：字号、正文宽度、深色模式、左右目录开关。
- `BookTableOfContents`：整本文档章节树与当前章节状态。
- `ArticleContent`
  - `MarkdownRenderer`：正文、GFM、代码复制、KaTeX、提示块。
  - `ChapterNavigation`：上一章、当前位置、下一章。
- 右栏
  - `ChapterTableOfContents`：正文标题锚点。
  - `ReadingProgressCard`：完成章节、进度、预计时间和下一节。

## 用户交互

- 目录和上一/下一章切换当前章节。
- 复制代码块写入剪贴板并短暂反馈“已复制”。
- 小/中/大字号、紧凑/舒适/宽正文和深色模式即时生效。
- 左右栏可独立折叠；阅读设置保存在 localStorage。
- 章节切换和滚动停止后保存阅读位置，重新打开时恢复；完成状态写入 SQLite。

## 页面状态

- 当前章节与当前标题高亮。
- 设置值：字号、正文宽度、主题、目录可见性。
- 代码复制：默认、成功。
- 章节边界：首章无上一篇，末章无下一章。

## 数据字段

文档标题与总进度、`ChapterNode`、`ChapterContent.content`、`ReadingProgress` 全字段；本章目录从 Markdown 标题派生。

## 响应式行为

宽屏三栏；中屏按用户设置隐藏右栏；窄屏左右栏作为可切换区域，正文占满可用宽度。表格、代码与公式局部横向滚动。

## 暂不实现

标注、高亮、评论、打印版、离线缓存、全文搜索、朗读和移动端手势。
